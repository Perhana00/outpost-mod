# Outpost Mod Structure

## Folder Hierarchy

```plaintext
/outpost-mod
├── assets/
│   └── (contains all the necessary textures, sounds, and other assets used in the mod)
├── configs/
│   └── (contains configuration files to customize gameplay and settings)
├── scripts/
│   └── (contains all script files that handle mod logic and functionality)
├── data/
│   └── (holds various data files, including loot tables, recipes, etc.)
├── lang/
│   └── (contains language files for localizations)
└── README.md
```  

## Usage of Directories

- **assets/**: This directory is crucial for the visual and audio elements of the outpost mod. It stores all textures, sounds, and other assets that will be loaded during the gameplay.
- **configs/**: The configuration files found here allow players and developers to customize their experiences. Adjustments to gameplay mechanics, difficulty settings, and mod-specific options can be modified in this folder.
- **scripts/**: All the essential logic and functionalities of the mod are coded here using the modding language. It includes scripts that dictate the mod's behavior during gameplay.
- **data/**: Data files in this directory handle various aspects such as loot, crafting recipes, and other essential gameplay mechanics.
- **lang/**: This directory is important for players worldwide, as it contains localization files to support different languages, ensuring accessibility to a broader audience.
